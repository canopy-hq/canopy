## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Pane header inline padding, splitter visual line offset |
| sm | 8px | Pane header vertical padding, gap between header elements |
| md | 16px | Pane header horizontal padding |
| lg | 24px | (reserved for future phases) |
| xl | 32px | (reserved for future phases) |

Exceptions:
- Splitter hit area: 6px wide (not a multiple of 4 -- intentional for precise pointer interaction)
- Splitter visual line: 2px wide (subpixel precision for thin divider)
- Minimum pane size: 120px width, 80px height (prevents unusable panes)

## Interaction Contracts

### Keyboard Shortcuts (KEYS-01, KEYS-02, KEYS-03)

| Shortcut | Action | Requirement |
|----------|--------|-------------|
| `Cmd+D` | Split focused pane horizontally (side by side) | TERM-02 |
| `Cmd+Shift+D` | Split focused pane vertically (top and bottom) | TERM-02 |
| `Cmd+W` | Close focused pane | TERM-06 |
| `Cmd+Option+ArrowLeft` | Navigate to pane left of current | TERM-05 |
| `Cmd+Option+ArrowRight` | Navigate to pane right of current | TERM-05 |
| `Cmd+Option+ArrowUp` | Navigate to pane above current | TERM-05 |
| `Cmd+Option+ArrowDown` | Navigate to pane below current | TERM-05 |

Passthrough rule: All other keystrokes propagate to focused xterm.js instance unchanged (KEYS-03).

Interception method: `document.addEventListener('keydown', handler, { capture: true })` + `term.attachCustomKeyEventHandler()` returning `false` for matched shortcuts (KEYS-02).

### Splitter Drag (TERM-03)

| Behavior | Spec |
|----------|------|
| Drag start | `pointerdown` on splitter, call `setPointerCapture` |
| Drag move | Update flex ratios in Zustand store on `pointermove` |
| Drag end | `pointerup` releases capture |
| Minimum pane ratio | 0.1 (10% of parent -- prevents collapse to zero) |
| Resize PTY debounce | 100ms after last `pointermove` (prevents flooding backend) |
| FitAddon.fit() | Called on `requestAnimationFrame` during drag |

### Pane Focus (TERM-05)

| Behavior | Spec |
|----------|------|
| Click to focus | `pointerdown` anywhere in pane sets `focusedPaneId` in Zustand |
| Focus on split | New pane receives focus after split |
| Focus on close | Adjacent sibling receives focus (prefer left/above) |
| Focus on last close | New terminal spawns and receives focus (never zero panes) |

## State Shape (Zustand)

Relevant to UI contract -- the renderer depends on this structure.

```typescript
// Pane tree lives in a single Zustand store
interface PaneTreeStore {
  root: PaneNode;
  focusedPaneId: string | null;
  splitPane: (paneId: string, direction: 'horizontal' | 'vertical', newPtyId: number) => void;
  closePane: (paneId: string) => void;
  setFocus: (paneId: string) => void;
  navigate: (direction: 'up' | 'down' | 'left' | 'right') => void;
  updateRatio: (branchId: string, index: number, delta: number) => void;
}
```

## WebGL Context Budget

| Property | Value |
|----------|-------|
| Max simultaneous WebGL contexts | 8 |
| Strategy | Attach WebGL addon only to visible panes; dispose on panes exceeding budget using LRU eviction (least recently focused) |
| Re-attach trigger | Pane becomes focused or visible after budget frees a slot |
| Fallback behavior | Terminal without WebGL does NOT render in xterm.js v6 -- must always re-attach before showing pane |

Source: 02-RESEARCH.md Pitfall 1. WebKit limit unverified; 8 is conservative safe default.
