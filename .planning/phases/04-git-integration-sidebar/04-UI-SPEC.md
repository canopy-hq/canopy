# Phase 4 — UI Design Contract

> Visual and interaction contract for the git sidebar, workspace tree, and create branch/worktree modal. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | react-aria-components 1.16.0 (Tree, Dialog, Modal, ModalOverlay, Button, Select) |
| Icon library | none (Unicode glyphs: branch icon U+2387, worktree diamond U+25C6, chevrons) |
| Font | Menlo, Monaco, "Courier New", monospace (inherited from existing app) |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap in tree items, inline padding |
| sm | 8px | Tree item vertical padding, compact button padding |
| md | 16px | Sidebar horizontal padding, modal section spacing |
| lg | 24px | Modal content padding, tree section gaps |
| xl | 32px | Modal header-to-body gap |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions:
- Sidebar default width: 230px (SIDE-01 requirement, not a spacing token)
- Sidebar min width: 180px, max width: 400px (resize bounds)
- Sidebar resize handle hit area: 4px wide (visual 1px border, 4px grab zone)
- Tree item indent per level: 16px
- Tree item height: 28px (compact for dense branch lists)
- Modal width: 480px (fixed, centered)

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 13px | 400 (regular) | 1.4 | Tree item labels, modal body text, type card labels |
| Small | 11px | 400 (regular) | 1.3 | Status bar text, ahead/behind badges, worktree path display, git command preview |
| Heading | 16px | 600 (semibold) | 1.2 | Modal title ("Create Branch or Worktree"), sidebar section headers |

Font family for all: `Menlo, Monaco, "Courier New", monospace` (matches existing app convention -- StatusBar, PaneHeader already use this).

Note: Phase 4 uses 3 type sizes (11px, 13px, 16px). The 2px gap between 11px and 13px is intentional -- 11px is reserved for secondary/metadata content (badges, paths, preview commands) while 13px is the primary reading size for tree item labels and modal body text. These roles never overlap, so the small numeric gap does not create visual ambiguity.

## Color

All colors use the existing CSS custom property system (`--bg-primary`, `--accent`, etc.) to stay theme-compatible across all 8 themes.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg-primary)` | App background, modal backdrop area |
| Secondary (30%) | `var(--bg-secondary)` | Sidebar background, modal surface |
| Tertiary | `var(--bg-tertiary)` | Tree item hover state, modal type card background, input backgrounds |
| Border | `var(--border)` | Sidebar right border, modal border, input borders, tree dividers |
| Accent | `var(--accent)` | Selected tree item indicator, active type card border, primary action button |
| Text primary | `var(--text-primary)` | Branch/worktree names, modal heading, focused item text |
| Text muted | `var(--text-muted)` | Ahead/behind counts, worktree paths, placeholder text, inactive items |

### Phase 4-Specific Colors (New Semantic Tokens)

These must be declared as CSS custom properties in the theme layer so they adapt across all 8 themes.

| Token | Default Value | CSS Custom Property | Usage |
|-------|---------------|---------------------|-------|
| Branch icon color | `#60a5fa` (blue-400) | `var(--branch-icon)` | Branch icon (U+2387) in sidebar tree items |
| Worktree icon color | `#c084fc` (purple-400) | `var(--worktree-icon)` | Worktree diamond (U+25C6) in sidebar tree items |
| Ahead badge | `#4ade80` (green-400) | `var(--git-ahead)` | Ahead count text (e.g., "+3") |
| Behind badge | `#f87171` (red-400) | `var(--git-behind)` | Behind count text (e.g., "-2") |
| Destructive | `#ef4444` (red-500) | `var(--destructive)` | Delete branch/worktree button, error states |

Accent reserved for: selected tree item left border indicator (2px), active type card border in modal, "Create Branch"/"Create Worktree" primary action button background, sidebar toggle keyboard hint.

---

## Focal Point

When the sidebar is visible, the **workspace tree** is the focal point of the sidebar panel. The tree occupies the largest area and contains the primary interactive elements. The user's eye should land on the expanded repository's branch list first, guided by the colored branch/worktree icons against the muted sidebar background. The "Import Repository" button at the bottom and the status bar updates are secondary; they support the tree but do not compete for attention.

When the create modal is open, the **name input field** is the focal point. It receives autofocus on open and is the first action the user takes after selecting a type card.

## Interaction Contracts

### Sidebar Toggle (Cmd+B)

| State | Behavior |
|-------|----------|
| Sidebar visible | Cmd+B hides sidebar instantly, terminal area expands to fill |
| Sidebar hidden | Cmd+B shows sidebar at last-used width |
| Initial state | Hidden (no repos imported yet) |

### Sidebar Resize

| State | Behavior |
|-------|----------|
| Mouse down on handle | Begin resize, set cursor to col-resize on document |
| Mouse move | Update sidebar width, clamped to 180-400px |
| Mouse up | End resize, persist width in workspace store |
| During resize | Terminal panes reflow via ResizeObserver (debounced, existing pattern) |

### Tree Item Selection

| Action | Behavior |
|--------|----------|
| Click repo header | Toggle expand/collapse |
| Click branch item | Select branch (highlight), update status bar |
| Click worktree item | Select worktree (highlight), update status bar |
| Right-click branch | No context menu for Phase 4 (deferred) |

### Import Repository Flow

| Step | Behavior |
|------|----------|
| Click "Import Repository" | Opens native macOS folder picker |
| User selects folder | IPC call to `import_repo`, on success: add to store, expand in tree, auto-select |
| Selected path is not a git repo | Error toast: "Not a git repository. Select a folder containing a .git directory." |
| User cancels picker | No action |

### Create Branch/Worktree Flow

| Step | Behavior |
|------|----------|
| Click "+ New Branch" in expanded repo | Open modal with that repo as context |
| Select type card | Toggle between Branch and Worktree, update preview and button label |
| Type name | Live-update git command preview |
| Select base branch | Dropdown of local branches from that repo |
| Click "Create Branch" / "Create Worktree" | IPC call to backend, on success: close modal, refresh tree, select new item |
| Backend error | Error toast with message (e.g., "Branch name already exists") |
| Click "Discard" or Esc | Close modal, no action |

### Delete Branch/Worktree

| Action | Behavior |
|--------|----------|
| Trigger | Not in Phase 4 requirements as a UI action -- GIT-02/GIT-03 specify backend CRUD. Add delete buttons only if time permits; otherwise defer to Phase 6 settings. |
| If implemented | Hover reveals trash icon, click shows confirmation: "Delete branch {name}? This cannot be undone." with Discard / Delete (red) buttons. |

## Layout Integration

### App.tsx Layout Change

Current layout (Phase 3):
```
TabBar
  PaneContainer (flex-1)
StatusBar
```

Phase 4 layout:
```
TabBar
  [Sidebar | PaneContainer] (flex-1, flex-row)
StatusBar
```

The middle row becomes a horizontal flex container. Sidebar sits left, PaneContainer fills remaining space. When sidebar is hidden, PaneContainer takes full width.
