    **2. Create `src/stores/theme-store.ts`:**

    ```typescript
    import { create } from 'zustand';
    import { type ThemeName, xtermThemes } from '../lib/themes';
    import { getAllCached } from '../lib/terminal-cache';

    interface ThemeState {
      currentTheme: ThemeName;
      setTheme: (theme: ThemeName) => void;
      initTheme: () => Promise<void>;
    }
    ```

    Implementation:
    - `currentTheme` defaults to `'obsidian'` (D-13).
    - `setTheme(theme)`:
      1. Update state: `set({ currentTheme: theme })`
      2. Apply to DOM: `document.documentElement.setAttribute('data-theme', theme)` (D-12)
      3. Apply to all cached xterm terminals: iterate `getAllCached()`, set `entry.term.options.theme = xtermThemes[theme]` (D-15)
      4. Persist: lazy-import `@tauri-apps/plugin-store`, call `(await import('@tauri-apps/plugin-store')).load('settings.json')` then `store.set('theme', theme)` then `store.save()`. Wrap in try/catch -- persistence failure should not block theme switch. In test environment (no Tauri runtime), the import will fail silently.
    - `initTheme()`:
      1. Try to load from store: `const store = await (await import('@tauri-apps/plugin-store')).load('settings.json')`, `const saved = await store.get<string>('theme')`.
      2. If saved and valid ThemeName, call `setTheme(saved)`.
      3. If not, call `setTheme('obsidian')` to apply default to DOM.
      4. Wrap in try/catch -- if Tauri not available (tests), just apply obsidian.
