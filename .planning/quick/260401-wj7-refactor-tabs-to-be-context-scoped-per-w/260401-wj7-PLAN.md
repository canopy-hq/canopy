<objective>
Each workspace item (branch/worktree) gets its own tab group. The tab bar shows only the active context's tabs. Switching sidebar items switches the entire tab group. Cmd+T creates a tab within the current context.

Strategy: keep flat `tabs[]` array but add `activeContextId` and `contextActiveTabIds` map. TabBar filters by context. Tab operations scope to context. Minimal disruption to existing pane operations.
</objective>

<tasks>

<task type="auto" tdd="true">
  <name>Task 1: Add context-scoped tab management to tabs-store</name>
  <files>
    src/stores/tabs-store.ts,
    src/stores/__tests__/tabs-store.test.ts
  </files>
  <read_first>
    src/stores/tabs-store.ts,
    src/stores/__tests__/tabs-store.test.ts
  </read_first>
  <behavior>
    **Data model changes:**
    - Add `activeContextId: string` field (default: `"default"`)
    - Add `contextActiveTabIds: Record<string, string>` (tracks last active tab per context)
    - Initial tab gets `workspaceItemId: "default"`
    - Add `setActiveContext(contextId: string, label?: string): void` action
    - Add `getContextTabs(): Tab[]` getter that returns `tabs.filter(t => t.workspaceItemId === activeContextId)`

    **setActiveContext behavior:**
    1. Save current activeTabId to contextActiveTabIds[current activeContextId]
    2. Set activeContextId = contextId
    3. Check if any tab has workspaceItemId === contextId
    4. If yes: restore activeTabId = contextActiveTabIds[contextId] (or first matching tab if no saved id)
    5. If no: create new tab with workspaceItemId=contextId and label=label??'Terminal', set as activeTabId, save to contextActiveTabIds

    **Modified operations (scope to active context):**
    - `addTab()`: new tab gets `workspaceItemId: state.activeContextId` (not undefined)
    - `closeTab(tabId)`: when closing last tab in context, spawn fresh tab with same workspaceItemId. Filter context tabs to check "last tab" condition.
    - `switchTabByIndex(index)`: index is relative to context tabs (filter first, then index)
    - `switchTabRelative(direction)`: wraps within context tabs only
    - Remove `findOrCreateTabForWorkspaceItem` (replaced by `setActiveContext`)

    **Unchanged operations (still use activeTabId globally):**
    - `splitPane`, `closePane`, `setFocus`, `navigate`, `updateRatio`, `setPtyId` — these operate on the active tab regardless of context, which is correct.
    - `switchTab(tabId)` — direct tab switch, still works globally
    - `getActiveTab()` — returns tab matching activeTabId, unchanged

    **Test updates:**
    - Update initial state tests: initial tab has workspaceItemId 'default'
    - Update addTab tests: new tabs get activeContextId
    - Add `describe('setActiveContext')` block:
      - Creates new tab group when switching to unknown context
      - Restores previous tab group when switching back
      - Remembers active tab per context
    - Add `describe('getContextTabs')` block:
      - Returns only tabs matching activeContextId
    - Update switchTabByIndex/switchTabRelative to work within context
    - Update closeTab: last tab in context spawns fresh context tab
    - Remove findOrCreateTabForWorkspaceItem tests
  </behavior>

  </verify>
  <done>
    tabs-store has context-scoped tab management. setActiveContext switches between isolated tab groups. Tab operations scope to active context.
  </done>
</task>
