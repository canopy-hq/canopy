  <action>
**In agent_watcher.rs — add output-based detection module:**

1. Create `OutputAgentDetector` struct holding per-PTY detection state:
   ```rust
   pub struct OutputAgentDetector {
       state: DetectionState,        // Idle | Running(agent_name, pseudo_pid)
       ring_buf: Vec<u8>,            // Rolling buffer for pattern matching (~512 bytes)
       ring_pos: usize,
   }
   
   enum DetectionState {
       Idle,
       Running { agent_name: String },
   }
   ```

2. Define agent output signatures as byte patterns (const arrays):
   - Claude Code: `b"claude>"`, `b"\xe2\x80\xba"` (the `›` prompt char Claude uses), `b"Claude Code"` (startup banner)
   - aider: `b"aider>"`, `b"Aider v"` (startup banner)
   - codex: `b"codex>"`, `b"Codex v"`
   - gemini: `b"gemini>"`, `b"Gemini"`
   
   Store as `AGENT_OUTPUT_PATTERNS: &[(&str, &[&[u8]])]` mapping agent name to its byte patterns.

3. Implement `OutputAgentDetector::new() -> Self` and `OutputAgentDetector::feed(&mut self, bytes: &[u8]) -> Option<AgentDetectionEvent>`:
   - Append bytes to ring buffer (overwrite oldest when full)
   - When in Idle state: scan ring buffer against all AGENT_OUTPUT_PATTERNS. On match, transition to Running, return `AgentDetectionEvent::Started { agent_name }`.
   - When in Running state: scan for shell prompt return patterns: `b"$ "`, `b"% "`, `b"# "`, `b"\n❯ "`, the typical zsh/bash prompt endings. On match, transition to Idle, return `AgentDetectionEvent::Stopped { agent_name }`.
   - Return None if no state change.

4. Add `AgentDetectionEvent` enum:
   ```rust
   pub enum AgentDetectionEvent {
       Started { agent_name: String },
       Stopped { agent_name: String },
   }
   ```

5. Add `OutputAgentDetector` storage to `AgentWatcherState`:
   ```rust
   pub detectors: HashMap<u32, OutputAgentDetector>,
   ```
   Initialize in `AgentWatcherState::new()`.

**In pty.rs — wire detector into reader thread:**

6. The reader thread (line 80-95) needs access to the detector and app_handle to emit events. Restructure `spawn_terminal`:
   - Before spawning reader thread, create the `OutputAgentDetector` and store it in `AgentWatcherState.detectors` for pty_id.
   - Clone a `Mutex<OutputAgentDetector>` (or use `Arc<Mutex<OutputAgentDetector>>`) and pass to reader thread.
   - Clone `app_handle` for the reader thread.
   - In the reader loop, after `reader.read(&mut buf)` succeeds, call `detector.feed(&buf[..n])`. On `Some(Started{..})`, emit `agent-status-changed` with status "running" and pid=0 (no real PID from output matching). On `Some(Stopped{..})`, emit with status "idle".

7. In `close_pty`, also remove the detector: `ws.detectors.remove(&pty_id)`.

**Keep kqueue as secondary:** The existing kqueue watcher in `start_watching` stays unchanged. If kqueue detects an agent (with a real PID), that's fine — the frontend store handles duplicate events gracefully (setAgent is idempotent). The output detector is just faster and more reliable.

**Important details:**
- The ring buffer approach avoids unbounded memory growth — 512 bytes is enough for any prompt pattern.
- Pattern matching must handle bytes split across reads (agent prompt might arrive in two chunks) — the ring buffer solves this.
- Use pid=0 for output-detected agents since we don't have the real PID. The frontend already handles pid=0 (manual toggle uses it).
- The silence timer still works: it reads last_output timestamps which are already updated in the reader thread.
