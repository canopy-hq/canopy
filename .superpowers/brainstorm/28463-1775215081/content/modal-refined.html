Git Worktree Rules (verified)

  • HEAD branch (checked out in main repo): Cannot create worktree → must create NEW branch -b name based on it
  • Local branch (not checked out, not in worktree): Can create worktree directly — moves the branch into the worktree
  • Branch already in a worktree: Cannot create another → show as "in use", offer to Open
  • Origin-only branch: Can create worktree directly (git checks it out locally)
  • Existing worktree: Already exists on disk → just Open (add to sidebar)
