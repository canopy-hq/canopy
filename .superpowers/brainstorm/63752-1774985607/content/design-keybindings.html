Design — Section 4: Keyboard Shortcuts & Pane Management

Terminal-native keybindings with full customization support

Pane Tree Model

Recursive binary tree — each node is either a split container or a terminal leaf. Supports arbitrary nesting.

PaneNode = Split { direction, children: [PaneNode, PaneNode], ratio }
│
└─ Leaf { pty_id, focused: bool }

Example: ⌘D then ⌘⇧D on right pane

Split(horizontal)
├─ Leaf(pty_1) ← left terminal
└─ Split(vertical)
├─ Leaf(pty_2) ← top-right
└─ Leaf(pty_3) ← bottom-right (focused)

Agent

All shortcuts passthrough to terminal by default.
App intercepts only when modifier combos match registered shortcuts.
Unmatched keys → PTY stdin as-is.

Keybinding Architecture

Keydown event
│
▼
KeybindingRegistry.match(event)
│
├─ matched → preventDefault + execute action
│
└─ no match → passthrough to focused xterm.js instance → PTY

Registry loads from:
1. Built-in defaults (hardcoded)
2. User overrides from ~/.superagent/keybindings.json
↳ User overrides win on conflict
